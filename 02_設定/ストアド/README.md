## ファイル

| ファイル | 呼出名 | 処理 |
|---|---|---|
| `SmxNyu.sql` / `SmxNyu.md` | `SmxNyu` | 登録本処理。通常QRは洗浄方法000以外で`T_SmxTrc`追加、000で`T_シートリーダ`追加。`$Ny`は元`T_SmxTrc`消込と`T_シートリーダ`追加 |
| `SmxNyuItem.sql` | `SmxNyuItem` | 通常QRは`T_data54`、`$Ny`は元連番を照合し、親品目・品名・数量・マスタ洗浄方法を返す |

## INPUT項目順

### `SmxNyu`

`SmxNyu`は既存の`Soc.YmdSend`形式を使用します。提示電文の例は次の形です。

```text
SmxNyu^000001^26/06/27^10:27:24^A999^2018/10/25^1000020880,P87303533,P95361443,TI_AS,66,2026/06/01,2026/05/27,,1KAF,,,^P95361443@P87303533^^55^0^0^005
```

| 受信位置 | ストアド引数 | 内容 |
|---:|---|---|
| 2 | `@bhtid` | BHTID |
| 3 | `@bhtdate` | BHT日付 |
| 4 | `@bhttime` | BHT時間 |
| 5 | `@syaincd` | 社員CD |
| 6 | `@dt` | 納入日。QR内処理日が取得できない場合の登録日付補助 |
| 7 | `@scandata` | QR全文。通常QRまたは`$Ny` QR |
| 8 | `@zno` | 通常QRでは`子品目@親品目` |
| 9 | `@rno` | 予備 |
| 10 | `@nyusuu` | 数量 |
| 11 | `@syusuu` | 予備 |
| 12 | `@zaisuu` | 予備 |
| 13 | `@biko` | 洗浄方法（`000`形式） |

### `SmxNyuItem`

Socketサーバーの`SmxNyuItem.ini`では、呼出名を除く受信項目を次の順にINPUTパラメータへ割り当てます。先頭6項目が他の山寺・SMX通信と共通で、以降がSMX戻り品入庫の個別項目です。

| 受信位置 | ストアド引数 | 内容 |
|---:|---|---|
| 2 | `@bhtid` | BHTID |
| 3 | `@bhtdate` | BHT日付 |
| 4 | `@bhttime` | BHT時間 |
| 5 | `@syaincd` | 社員CD |
| 6 | `@nonyudate` | 納入日 |
| 7 | `@qr` | QR全文 |
| 8 | `@destination` | 出庫先 |
| 9 | `@original_seq` | `$Ny`は元の連番、通常QRはQR内の処理日 |
| 10 | `@child_item` | 子品目 |
| 11 | `@quantity` | 数量 |
| 12 | `@cleaning_method` | 洗浄方法 |
| 13 | `@biko` | 親品目（通常QR・`$Ny`ともBHTから送信） |

## OUTPUT項目順

| 呼出名 | OUTPUT |
|---|---|
| `SmxNyu` | `@result`、`@msg` |
| `SmxNyuItem` | `@result`、`@msg`、`@parent_item`、`@item_name`、`@original_qty`、`@master_cleaning` |

Socketサーバー側では、OUTPUTの後ろにストアドのRETURN値が付加されます。`@result=0`が正常、エラー時は0以外の正の整数を返します。ストアドのRETURN値も正常は0、エラーは`@result`と同じ正の整数にします。`@msg`はハンディターミナルの小さい画面に表示するため、漢字10文字・半角20文字程度までの短文にします。

## result・msg方針

| 項目 | 方針 |
|---|---|
| 正常 | `@result=0` |
| エラー | `@result`へ0以外の正の整数を設定 |
| ストアドRETURN値 | 正常は`RETURN 0`、エラーは`RETURN @result` |
| msg長さ | 漢字10文字・半角20文字程度まで |
| msg用途 | BHT画面表示用。詳細原因はSQLコメント・MD・ログ側で確認する |

## resultコード

意味の同じエラーは、各ストアドで同じ番号を返します。

| result | msg | 意味 |
|---:|---|---|
| 0 | `OK`、`入庫登録`、管理番号 | 正常 |
| 1 | `QR不正`、`元連番不正`、`元連番なし`、`品名なし` | 照会対象なし・照会キー不正 |
| 2 | `商品が違います` | 子品目・親子品目の不一致または不足 |
| 3 | `登録済` | すでに終了・登録済み |
| 4 | `数量不正` | 数量が未入力、数値以外、または1未満 |
| 5 | `洗浄不正` | 洗浄方法の入力値不正 |
| 6 | `出庫先が違います` | 出庫先が`1000020880`ではない、または元レコードの出庫先と一致しない |
| 9 | `品名照会失敗`、`入庫登録失敗` | SQL例外・登録更新失敗 |

## DB処理

| 処理 | 内容 |
|---|---|
| 商品照会 | 通常QRは`T_data54`、`$Ny`は`T_SmxTrc.連番`で照合する |
| 品名取得 | `T_data54.親図面番号=上位品目`かつ`T_data54.図番=子品目@上位品目`で検索し、品名と洗浄方法を返す |
| 通常QRのマスタ未登録 | `T_data54`に該当データがなければ、結果`1`・「品名なし」を返し、入庫入力へ進まない |
| 洗浄方法の初期値 | 通常QRの該当データがある場合、`T_data54.洗浄方法`をBHTの洗浄方法入力初期値として返す |
| 通常QR・洗浄方法000以外 | `SmxNyu`で`T_SmxTrc`へ追加し、`T_シートリーダ`へは追加しない。正常時の`@msg`は採番した管理番号 |
| 通常QR・洗浄方法000 | `SmxNyu`で`T_シートリーダ`へ追加し、`T_SmxTrc`へは追加しない |
| `$Ny` QR | `SmxNyu`で元`T_SmxTrc.終了日時`を更新し、`T_シートリーダ`へ追加する。洗浄方法が000以外でもシート追加する |
| `T_SmxTrc`採番 | `ID`はIDENTITY主キーのためINSERTしない。`連番`はBHTへ返す管理番号として`MAX([連番])+1`で採番する |
| `T_SmxTrc`必須列 | `自IPアドレス`は利用者指定により社員NOを設定し、`現在日付`はBHT日付、`BHT側時刻`はBHT時刻を設定する |
| 同時実行制御 | `$Ny`は元連番、通常QRのシート登録はQR単位、通常QRの`T_SmxTrc`追加は`T_SmxTrc_連番`の`sp_getapplock`で直列化する。`$Ny`だけは同じQRのシート再送を二重登録しない |

登録可能な出庫先はWeb版の既定値に合わせ、各ストアド内の`@allowed_destination`を`1000020880`としています。環境が異なる場合は適用前に変更してください。

## Socketサーバー側の設定

Socketサーバー用INIは次の場所へ追加済みです。

| 呼出名 | INI |
|---|---|
| `SmxNyu` | `ソケットサーバ/ini/SmxNyu.ini` |
| `SmxNyuItem` | `ソケットサーバ/ini/SmxNyuItem.ini` |

既存`SmxNyu.ini`に合わせ、接続先は`Data Source=.;Initial Catalog=tksSmx;Integrated Security=True;`、文字コードはCP932です。配置先環境で接続方式が異なる場合は変更してください。
