## 目的

`SmxNyu`はSMX入庫の登録本処理です。通常QRと`$Ny`戻り品QRを同じ呼出名で受け取り、QR種別と洗浄方法により登録先を分岐します。

| QR種別 | 条件 | 処理 |
|---|---|---|
| 通常QR | 洗浄方法が`000`以外 | `T_SmxTrc`へ追加し、`T_シートリーダ`へは追加しない |
| 通常QR | 洗浄方法が`000` | `T_シートリーダ`へ追加し、`T_SmxTrc`へは追加しない |
| `$Ny` QR | 洗浄方法に関係なし | 元`T_SmxTrc.終了日時`を更新し、`T_シートリーダ`へ追加する |

## 受信電文

`Soc.YmdSend`の既存形式で受信します。

```text
SmxNyu^BHTID^BHT日付^BHT時刻^社員CD^納入日^QR全文^子品目@親品目^^数量^0^0^洗浄方法
```

例:

```text
SmxNyu^000001^26/06/27^10:27:24^A999^2018/10/25^1000020880,P87303533,P95361443,TI_AS,66,2026/06/01,2026/05/27,,1KAF,,,^P95361443@P87303533^^55^0^0^005
```

## INPUT

| 受信位置 | 引数 | 内容 |
|---:|---|---|
| 2 | `@bhtid` | BHTID |
| 3 | `@bhtdate` | BHT日付 |
| 4 | `@bhttime` | BHT時刻 |
| 5 | `@syaincd` | 社員CD |
| 6 | `@dt` | 納入日。QR内処理日が取得できない場合の補助日付 |
| 7 | `@scandata` | QR全文 |
| 8 | `@zno` | 通常QRでは`子品目@親品目` |
| 9 | `@rno` | 予備 |
| 10 | `@nyusuu` | 数量 |
| 11 | `@syusuu` | 予備 |
| 12 | `@zaisuu` | 予備 |
| 13 | `@biko` | 洗浄方法。`000`形式 |

## QR解析

| QR種別 | 判定 | 主な取得項目 |
|---|---|---|
| `$Ny` QR | `@scandata`の先頭3文字が`$Ny` | QR 2項目目=出庫先、3項目目=元連番、4項目目=子品目、5項目目=数量、6項目目=洗浄方法 |
| 通常QR | `$Ny`以外 | QR 1項目目=出庫先、4項目目=棚番、6項目目=所要日、7項目目=処理日、9項目目=プラント、10項目目=伝票番号、11項目目=ロット、12項目目=保管場所、`@zno`=子品目@親品目 |

通常QRで`@zno`に`@`がない場合は、QR 2項目目を親品目、3項目目を子品目として扱います。

## 入力チェック

| result | msg | 条件 |
|---:|---|---|
| 1 | `QR不正` | QRをカンマ分割用XMLへ変換できない |
| 4 | `数量不正` | 数量が数値でない、または1未満 |
| 5 | `洗浄不正` | 洗浄方法が数値でない、または負数 |
| 6 | `出庫先が違います` | 出庫先が`1000020880`ではない |
| 2 | `商品が違います` | 子品目が空、または通常QRで親品目が空 |

## 通常QR・洗浄方法000

| 項目 | 内容 |
|---|---|
| 登録先 | `T_シートリーダ` |
| 排他 | `sp_getapplock`、リソース`T_SmxTrc_通常入庫_` + QRチェックサム |
| `T_SmxTrc` | 追加しない |
| 正常応答 | `result=0`、`msg=入庫登録`、`RETURN 0` |

`T_シートリーダ`の登録値:

| 列 | 値 |
|---|---|
| `bcd` | QR全文 |
| `dt` | QR内処理日。なければ`@dt` |
| `kbn` | `2` |
| `biko` | 空文字 |
| `図番累進` | 子品目 + `@` + 親品目 |
| `記号` | `SMX入庫` |
| `NO` | `0` |
| `数` | 数量 |
| `メモ` | `SMX` |
| `図番` | 子品目 + `@` + 親品目 |
| `場所` | 空文字 |

## 通常QR・洗浄方法000以外

| 項目 | 内容 |
|---|---|
| 登録先 | `T_SmxTrc` |
| 排他 | `sp_getapplock`、リソース`T_SmxTrc_連番` |
| 採番 | `MAX([連番]) + 1` |
| `T_シートリーダ` | 追加しない |
| 正常応答 | `result=0`、`msg=採番した連番`、`RETURN 0` |

`T_SmxTrc`の登録値:

| 列 | 値 |
|---|---|
| `ID` | 登録しない。IDENTITY主キーのためDBが採番 |
| `自IPアドレス` | `@syaincd`の先頭15文字。今回の電文では社員NOを使用 |
| `現在日付` | `@bhtdate` |
| `BHT側時刻` | `@bhttime` |
| `社員CD` | `@syaincd` |
| `出庫先CD` | QR 1項目目 |
| `出庫品目` | 子品目 |
| `上位品目` | 親品目 |
| `棚番` | QR 4項目目 |
| `数量` | 数量 |
| `所要日` | QR 6項目目 |
| `処理日` | QR 7項目目。なければ`@dt` |
| `プラント` | QR 9項目目 |
| `伝票番号` | QR 10項目目 |
| `ロット` | QR 11項目目 |
| `保管場所` | QR 12項目目 |
| `洗浄方法` | 3桁ゼロ埋め洗浄方法 |
| `QRコード` | QR全文 |
| `連番` | `MAX([連番]) + 1` |
| `開始日時` | `GETDATE()` |
| `終了日時` | `NULL` |
| `備考` | `@rno` |
| `tm` | 登録しない。既定値`GETDATE()` |

## `$Ny` QR

| 項目 | 内容 |
|---|---|
| 対象 | QR内の元連番に一致する最新の`T_SmxTrc` |
| 排他 | `sp_getapplock`、リソース`T_SmxTrc_入庫_元連番` |
| 更新 | 元`T_SmxTrc.終了日時 = GETDATE()` |
| 登録 | `T_シートリーダ`へ追加 |
| 洗浄方法 | `000`以外でも`T_シートリーダ`へ追加する |
| 再送対応 | 同じQR・`記号=SMX入庫`が既にある場合、シート登録は重複追加しない |
| 正常応答 | `result=0`、`msg=入庫登録`、`RETURN 0` |

追加検証:

| result | msg | 条件 |
|---:|---|---|
| 1 | `元連番不正` | QR内元連番が数値でない |
| 1 | `元連番なし` | 元連番に一致する`T_SmxTrc`がない |
| 6 | `出庫先が違います` | QR出庫先と元レコード出庫先が一致しない |
| 2 | `商品が違います` | QR子品目と元レコード出庫品目が一致しない |
| 3 | `登録済` | 元レコードの`終了日時`が設定済み |

## 例外処理

| 項目 | 内容 |
|---|---|
| ログ出力 | `dbo.t_log`へ受信内容を記録する。失敗しても本処理は止めない |
| トランザクション | DB更新はトランザクション内で実行する |
| 例外時 | `ROLLBACK`し、`result=9`、`msg=入庫登録失敗`、`RETURN 9` |
| msg方針 | BHT画面向けに短文とする |

## 判断理由

| 判断 | 理由 |
|---|---|
| 登録本処理を`SmxNyu`へ集約 | BHTから提示電文形式で送信されるため |
| 通常QRの洗浄方法000は`T_SmxTrc`へ追加しない | 洗浄なしの入庫実績として`T_シートリーダ`だけに残すため |
| 通常QRの洗浄方法000以外は`T_シートリーダ`へ追加しない | 洗浄出庫として`T_SmxTrc`側で管理するため |
| `$Ny`は洗浄方法に関係なくシート追加 | 戻り品の消込と入庫実績を同時に残すため |
| `ID`ではなく`連番`を返す | `ID`はDB内部主キー、`連番`はBHTへ表示する管理番号として使うため |
| `自IPアドレス`に社員NOを設定 | 現行`SmxNyu`電文にIP項目がなく、利用者指定により社員NOを必須列へ格納するため |
| `sp_getapplock`を使用 | 連番採番、同一QR登録、同一元連番更新の競合を防ぐため |
