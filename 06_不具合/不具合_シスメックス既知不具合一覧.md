## 2. 未対応の既知不具合

### BUG-SMX-001 Wi-Fi無効時に未送信でも正常扱いになる

| 項目 | 内容 |
|---|---|
| 現象 | Wi-Fi無効状態でSMX登録を操作すると、DBへ送信していないのにBHTが正常と判定し得る |
| 原因 | `Soc.YmdSend`が`SysWifi=false`の場合にSocket通信を行わず`0`を返す |
| 影響範囲 | SMX通常QR入庫、`$Ny`戻り品入庫、SMX出荷、対象BHT利用者 |
| 回避方法 | 起動時LIVE通信が正常な無線有効端末だけで操作し、登録後にDB側を確認する |
| 状況 | 未対応 |
| 関連テスト | `COM-06` |

### BUG-SMX-002 送信後の応答欠落で二重登録の可能性がある

| 項目 | 内容 |
|---|---|
| 現象 | サーバー登録後に応答だけが失われると、次回再送時に同じ処理が重複する可能性がある |
| 原因 | 受信失敗時は`sp-send.txt`に電文が残り、新しい操作でも電文を追加する。共通電文に一意な取引IDがない |
| 影響範囲 | 通常QRの既存`SmxNyu`、SMX出荷`SmxSyu`。`$Ny`の新SQLは元連番・QRで重複抑止を行う |
| 回避方法 | 通信異常後は再操作前にDB登録結果と`sp-send.txt`の残件を確認する |
| 状況 | 未対応 |
| 関連テスト | `COM-03`、`IN-R-09` |

### BUG-SMX-003 Socket接続失敗が正常扱いになる可能性がある

| 項目 | 内容 |
|---|---|
| 現象 | Socket接続で例外が発生しても、呼出元が接続成功と判定し得る |
| 原因 | `Sock.Conn`の例外経路で非0の戻り値を確実に設定しない |
| 影響範囲 | SMXのすべてのSocket呼出しと対象BHT利用者 |
| 回避方法 | 業務開始前にLIVE疎通を確認し、登録後はDB側の結果を照合する |
| 状況 | 未対応 |

### BUG-SMX-004 受信失敗時に自動再試行しない

| 項目 | 内容 |
|---|---|
| 現象 | 送信成功後に受信だけ失敗すると、応答受信を再試行せずエラー終了する |
| 原因 | `Soc.YmdSend`の再試行対象が`Sock.Send`エラーに限られている |
| 影響範囲 | SMX通常QR入庫、`$Ny`戻り品入庫、SMX出荷 |
| 回避方法 | 受信エラー後は、再操作する前にDB登録済みかを確認する |
| 状況 | 未対応 |
| 関連テスト | `COM-03` |

### BUG-SMX-005 前回のSocket応答が残る可能性がある

| 項目 | 内容 |
|---|---|
| 現象 | 受信失敗後に、前回通信の結果・メッセージを参照する可能性がある |
| 原因 | 実データ受信時以外は`Sock.Recv.Items$()`をクリアしない |
| 影響範囲 | SMXの結果コード、メッセージ、品名・洗浄方法表示 |
| 回避方法 | 通信異常後はBHTアプリを再起動し、前回応答が残らない状態で再確認する |
| 状況 | 未対応 |

### BUG-SMX-006 長いSMX電文が切れる可能性がある

| 項目 | 内容 |
|---|---|
| 現象 | 長いQRを含むSMX電文で、後半項目が欠落または切断される可能性がある |
| 原因 | `Sock.Param$`と`sp-send.txt`の1レコードが255文字固定で、送信前の長さ確認がない |
| 影響範囲 | QR全文を送信するSMX入庫、SMX出荷の長い電文 |
| 回避方法 | 長いQRの使用を避け、BHT送信電文とSocketサーバー受信値を照合する |
| 状況 | 未対応 |
| 関連テスト | `COM-07` |

### BUG-SMX-007 SMX出荷で社員／納入日入力をキャンセルしても次画面へ進む

| 項目 | 内容 |
|---|---|
| 現象 | SMX出荷の社員／納入日入力でM2を押しても、メニューに戻らず出荷票入力へ進む |
| 原因 | `gSmxSyu$`が`gSyacd$`の戻り値を確認せず、直後に`syori$ = "初期画面"`で上書きする |
| 影響範囲 | SMX出荷画面とその利用者 |
| 回避方法 | キャンセル後は出荷登録を行わず、M1でメニューへ戻る |
| 状況 | 未対応 |
| 関連テスト | `OUT-05` |

### BUG-SMX-008 不正な日付・時刻を受理する可能性がある

| 項目 | 内容 |
|---|---|
| 現象 | 非数値、存在しない日付、24時以上、60分以上の値を受理し得る |
| 原因 | 納入日・時刻の確認が先頭文字と文字数中心で、数値範囲や実在日を検証しない |
| 影響範囲 | SMX入庫・出荷の社員／納入日入力、SMX出荷票の出荷日 |
| 回避方法 | 正規発行のQRのみ使用し、BHT表示の日付・時刻を目視確認する |
| 状況 | 未対応 |

### BUG-SMX-009 通常SMX入庫QRの帳票識別が弱い

| 項目 | 内容 |
|---|---|
| 現象 | SMX入庫用ではないカンマ区切りQRを通常SMX入庫QRとして誤認する可能性がある |
| 原因 | 通常QR分岐は11項目以上と必須項目・数量を確認するが、業務識別子を確認しない |
| 影響範囲 | SMX通常QR入庫のQR読取と利用者 |
| 回避方法 | SMX入庫用に正規発行されたQRのみ使用し、表示された親・子品目を確認する |
| 状況 | 未対応 |

## 3. 対応済みの不具合

| ID | 現象 | 原因 | 影響範囲 | 対応内容 | 現在状況 |
|---|---|---|---|---|---|
| FIX-SMX-001 | 山寺・SMX送信例外時に別関数の終了先へ復帰する | `socYmdSend.Err`が`resume socSend.End`を指定 | 山寺入庫、SMX入庫、SMX出荷 | `resume socYmdSend.End`へ修正 | ソース修正済み、実機未確認 |
| FIX-SMX-002 | SMX入庫で社員／納入日キャンセル後も次画面へ進む | `gSyacd$`の戻り値を上書き | SMX入庫 | 戻り値が空でなければ入庫を終了するよう修正 | ソース修正済み、実機未確認 |
| FIX-SMX-003 | 通常QR・洗浄方法0で`T_シートリーダ`実績が作成されない | BHTが入力値0をシート登録経路へ分岐していなかった | SMX通常QR入庫 | 現行は`SmxNyu`内で洗浄方法000を判定し、`T_シートリーダ`へ追加する | ソース・SQL修正済み、適用・実機確認待ち |
| FIX-SMX-004 | `$Ny`・洗浄方法0で終了更新だけが残る可能性がある | BHT側の別通信で終了更新とシート登録を行うと一括ロールバックできない | `$Ny`戻り品入庫 | 現行は`SmxNyu`内で終了更新とシート登録を同一トランザクション実行 | SQL修正済み、適用・実機確認待ち |
| FIX-SMX-005 | 通常QRの品目マスタがなくても入庫入力へ進む | `SmxNyuItem`が`T_data54`未登録時も正常応答していた | SMX通常QR入庫 | 未登録時は結果`1`・「品名なし」とする | SQL修正済み、適用・実機確認待ち |
| FIX-SMX-006 | 通常QR・洗浄方法000以外の`T_SmxTrc`追加でSQLエラーになる | 実スキーマのNOT NULL列`自IPアドレス`・`現在日付`・`BHT側時刻`をINSERTしていなかった | SMX通常QR入庫のうち洗浄方法000以外の登録。洗浄方法000のシート登録、`$Ny`消込は直接影響しない | `SmxNyu.sql`のINSERT列を実スキーマに合わせ、`自IPアドレス`へ社員NO、BHT日付・時刻、QR明細列を登録する | SQL修正済み、適用・実機確認待ち |

## 4. 仕様確認中・保留事項

| ID | 現象 | 現行動作 | 影響範囲 | 暫定回避方法 | 確認事項 |
|---|---|---|---|---|---|
| HOLD-SMX-001 | `$Ny` QRの数量が元`T_SmxTrc.数量`と異なる | 「数量違い」と表示するがENTで登録続行できる | `$Ny`戻り品入庫と対象データ | 数量違いの表示時は登録せず、元データとQRを確認する | 警告のみで良いか、登録を禁止するか |
| HOLD-SMX-002 | 同じ通常QR・洗浄方法0を再送する | `T_シートリーダ`へ再度追加できる | SMX通常QR入庫と対象DB実績 | 通信異常後はDBを確認し、登録済みなら再操作しない | 同じQRの別作業再使用を優先するか、二重登録防止を優先するか |

## 5. 判断理由

| 判断 | 理由 |
|---|---|
| SMX専用の一覧を分離する | 全機能の不具合MDからSMXに関係する項目だけを運用・テスト時に参照しやすくするため |
| 対応済みも残す | 配布、SQL適用、実機テストが未実施で、環境上は旧動作が残っている可能性があるため |
| 仕様未確定は不具合と断定しない | 業務判断によって正常仕様または不具合のどちらにもなり得るため |
