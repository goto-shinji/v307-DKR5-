## 2. 結論

Bluetooth通信には、実際には失敗しているのに呼出元へ `0=正常` を返し得る経路がある。特に次の問題が重大である。

1. `BtCheck` は `St.Open` 失敗時に戻り値へエラーを設定しない。
2. `st.Fl2Pr` は定義ファイルエラー、ACKタイムアウト、NAK終了などを成功扱いし得る。
3. PT208詳細設定は空の `filename$(1)` を印刷処理へ渡す。
4. `St.setAdr` 失敗後も初期設定を続行し、後続結果でエラーを上書きし得る。

## 3. 通信方式

BluetoothはPT208/PW208プリンタ専用で、SPP接続文字列は次の形式である。

```text
COM4:M,SPP,<BDアドレス>,,30,1
```

無線LANのTCPソケット・FTP通信とは別経路である。

## 4. Bluetooth設定値

設定は `LifePd.DAT` を使う `Sts` 機構へ保存される。

| 設定 | 内容 | 初期値 |
|---|---|---:|
| `.BT.ADR$` | プリンタBDアドレス（12桁） | 空 |
| `.PRINTER$` | `PT208` / `PW208` | `PW208` |
| `.BT.SPD$` | 印字速度 | 3 |
| `.BT.DNS$` | 濃度 | 5 |
| `.BT.HGT$` | ラベル縦 | 1280 |
| `.BT.WID$` | ラベル横 | 384 |
| `.BT.PWR$` | 自動電源OFF | 120分 |
| `.BT.POX$`, `.BT.POY$` | 印刷位置補正 | 0 |
| `.BT.BTN$` | プリントキー設定 | 1 |
| `.PW.OFFSET$` | PW208ハクリ量 | 8 |
| `.PW.SLEEP1$` | PW208ライトスリープ | 5秒 |
| `.PW.SLEEP2$` | PW208ディープスリープ | 300分 |

プリンタ機種を変更した場合は `BtInit` が機種別初期値へ入れ替える。

## 5. プリンタ機種判定

BDアドレスへ接続してENQを送信し、応答長だけで機種を判定する。

| 受信長 | 判定 |
|---:|---|
| 11 | PT208 |
| 27または28 | PW208 |
| その他 | 判定失敗 |

応答内容や機種IDは確認していない。

## 6. プリンタ選択手順

### 6.1 BDアドレスを直接入力・スキャン

処理経路は `gBtIn$` → `BtCheck` → `St.Open` → `St.setAdr`。

| 手順 | 操作・処理 | チェック内容 |
|---:|---|---|
| 1 | 保守業務からプリンタ選択を開く | 現在のBluetooth接続を `St.Close` する |
| 2 | BDアドレスを手入力またはスキャン | 手入力は12桁確認。バーコード読取は12桁・16進文字を確認しない |
| 3 | `BtCheck(アドレス)` を呼ぶ | `St.Open` でSPP接続する |
| 4 | `St.setAdr` がENQを送る | `St.RcvData` が最大約1秒応答を待つ |
| 5 | 受信長でPT208/PW208を判定 | 機種とBDアドレスを保存する |
| 6 | 機種別初期設定を送る | PT208は `P00.dat`、PW208は設定コマンドを送る |

### 6.2 周辺プリンタを検索

処理経路は `gBtSrch$` → `gBtList$` → `bt.Scan` → `BtCheck`。

| 手順 | 操作・処理 | チェック内容 |
|---:|---|---|
| 1 | プリンタ検索を開始 | 現在のプリンタ接続を切る |
| 2 | `OPEN "COM4:I,5,5"` で探索 | 5秒、最大5台を検出する |
| 3 | `BT.FN3 10` でアドレス取得 | BDアドレスだけを表示し、機種名・電波強度は表示しない |
| 4 | 一覧から1台選択 | 周囲のプリンタ以外のBluetooth機器も選択できる |
| 5 | `BtCheck` を実行 | 接続・機種判定・保存・初期設定を行う |

## 7. ラベル印刷通信手順

処理経路は各印刷画面 → `St.Print` → `st.Fl2Pr`。

### 7.1 印刷前処理

| 手順 | 処理 | チェック内容 |
|---:|---|---|
| 1 | `St.Items$(1～9)` に図番・品名等を設定 | 呼出画面ごとに印刷項目を設定する |
| 2 | ラベル定義ファイル名を渡す | `PTenPr.DAT`、`PYmdPr.DAT`、`FEED.DAT` 等 |
| 3 | `Lbl.Open` で定義ファイルを開く | ファイル存在と空でないことを確認する |
| 4 | `St.Open` で保存済みBDアドレスへ接続 | 既オープン `&h37` は無条件に正常扱いする |

### 7.2 ラベルデータ送信

| 手順 | 送信内容 | 処理 |
|---:|---|---|
| 1 | STX | 印刷電文開始 |
| 2 | 定義ファイル各行 | 制御条件に合う行だけを採用する |
| 3 | `{@1}`～`{@9}` | `St.Items$()` の値で置換する |
| 4 | ESC + コマンド | プリンタコマンドを送る |
| 5 | ETX | 印刷電文終了 |

### 7.3 ACK/NAK・ステータス確認

| 手順 | 処理 | 判定 |
|---:|---|---|
| 1 | `St.RcvAck` が最大約5秒待つ | ACK、NAK、タイムアウト |
| 2 | ACKならENQ送信 | 印刷ステータスを照会する |
| 3 | NAKなら機種別NAK設定を参照 | 終了または再照会する |
| 4 | `St.RcvData` が最大約1秒待つ | STX位置から3文字後をステータスとする |
| 5 | PT/PW別ステータス表を参照 | 0=正常、1=表示して再照会、2=エラー終了 |

印刷後は明示的に切断せず、次の印刷でもCOM4接続を再利用する。

## 8. 不具合・不整合

### 8.1 重要度: 高

| 問題 | 関連機能 | 現在の動作 | 影響 |
|---|---|---|---|
| 接続失敗を成功として返す | `BtCheck`、`St.Open` | `St.Open` 失敗時、`result` にエラーを代入せず終了する | 誤アドレス、電源OFF、圏外でもプリンタ選択成功に見える |
| 印刷失敗でも0を返し得る | `St.Print`、`st.Fl2Pr` | 定義ファイルなし・空、ACKタイムアウト、NAK終了等で `result` を非0にしない | 未印刷でも成功扱いになる |
| PT208詳細設定が空ファイル名を使う | `gPset$`、`St.Print` | `St.Print("P00.dat",0,filename$)` の `filename$(1)` が空。`St.Print` は第1引数を無視する | `P00.dat` が送られず、さらに成功扱いになり得る |
| アドレス設定失敗後も続行 | `BtCheck`、`St.setAdr` | エラー表示後も機種別初期化へ進み、後続結果でエラーを上書きし得る | 接続・機種判定失敗が隠れる |

### 8.2 重要度: 中

| 問題 | 関連機能 | 影響 |
|---|---|---|
| 応答長だけで機種判定 | `St.setAdr` | 途中受信、余分なバイト、別機器応答で誤判定し得る |
| スキャンしたBDアドレスを未検証 | `gBtIn$` | 12桁以外・非16進文字列を接続処理へ渡し得る |
| 既オープンを無条件に正常扱い | `St.Open` | 接続中相手が保存アドレスと異なっても別プリンタへ送信し得る |
| PW208初期設定の応答未確認 | `BtPw208Init` | 設定コマンドが受理されたか確認できない |
| 印刷画面が戻り値を使わない | `pTenPr`、`pGenPr`、`dFeed` | 戻り値を正しても画面側で成功・失敗を通知できない |

### 8.3 重要度: 低または仕様確認

| 問題 | 関連機能 | 内容 |
|---|---|---|
| 品名チェック対象の誤り | `pTenPr` | 品名ではなく再度 `gtn.scd$` を確認している |
| タイムアウト固定 | `St.RcvData`、`St.RcvAck` | 約1秒・5秒を設定変更できない |
| 接続を保持 | `st.Fl2Pr` | 高速化できるが、交換・電源断後の再接続条件が複雑になる |
| 検索例外を表示しない | `gBtList$` | エラー内容を示さずメニューへ戻る |

## 9. 推奨改修順序

本書は調査文書であり、ソース改修は行っていない。

1. `BtCheck` の `St.Open` エラーを必ず戻り値へ反映する。
2. `st.Fl2Pr` の全異常終了で非0を返し、印刷画面でも結果を表示する。
3. PT208詳細設定で `filename$(1) = "P00.dat"` を設定するか、`St.Print` の引数仕様を整理する。
4. `St.setAdr` エラー時は即終了する。
5. 機種・アドレスは応答検証および初期設定成功後に確定する。
6. 応答内容・機種ID・BDアドレスを検証する。

## 10. 実機確認項目

| 試験 | 期待結果 |
|---|---|
| 存在しないBDアドレスを入力 | 選択失敗となり、旧設定を保持する |
| 11/27/28文字を返す非対象機器 | プリンタ機種として登録しない |
| プリンタ電源OFFで印刷 | 非0エラーとなり、成功音を出さない |
| ACKを返さないプリンタ | タイムアウトを呼出画面へ返す |
| NAK、紙切れ、カバー開 | 状態別表示後、復旧時だけ再照会する |
| PT208詳細設定 | `P00.dat` が送られ、設定反映を確認できる |
| PW208詳細設定 | ACKまたは設定読戻しで反映を確認できる |
