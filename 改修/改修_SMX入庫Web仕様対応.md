# SMX入庫 Web仕様対応

## 文書区分

| 項目 | 内容 |
|---|---|
| MD分類 | 改修 |
| 実施日 | 2026-06-21 |
| 対象 | BHTのSMX入庫、Socket通信、SQL Serverストアド |

## 調査内容

| 調査対象 | 確認結果 |
|---|---|
| BHTの従来SMX入庫 | カンマ区切りQRから親・子・数量を取得し、`SmxNyu`を1回送信していた |
| 既存共通電文 | `Soc.YmdSend`が呼出名、BHTID、BHT日付、BHT時間、社員CD、納入日を先頭に付加する |
| Socket応答 | `^`分割後、位置0が結果、位置1がメッセージ、位置2以降が追加OUTPUT |
| 通常QR | 品目マスタの洗浄方法を初期表示し、BHTで確定した入力値が0ならシートリーダへ登録する |
| `$Ny`仕様 | 6項目固定。数量・洗浄方法は表示のみとし、洗浄方法0は終了日時更新とシートリーダ登録、0以外は終了日時更新だけを行う |

| `_Org/gSmxNyu.src` | 通常QRだけに数量・洗浄方法入力画面を追加し、`$Ny`はQRの数量・洗浄方法を表示のみとするよう修正 |

## 通信仕様

| 呼出名 | 用途 | 正常時の主なOUTPUT |
|---|---|---|
| `SmxNyuItem` | 品名問い合わせ・登録前確認 | 結果、メッセージ、親品目、品名、元数量、マスタ洗浄方法 |
| `SmxNyuSheet` | 通常QRのシート登録、または`$Ny`洗浄方法0の終了更新＋シート登録 | 結果、メッセージ |
| `SmxNyuFinish` | `$Ny`洗浄方法0以外の`T_SmxTrc`終了日時更新 | 結果、メッセージ |

## 判断理由

| 判断 | 理由 |
|---|---|
| 3通信で同じINPUT形を使用 | Socketサーバー設定とBHT側送信処理を共通化し、項目ずれを防ぐため |
| 登録前にストアド側で再検証 | 品名照会後から登録までの間に他端末が処理する競合を防ぐため |
| 通常QRはBHT入力値で判断 | QR値やマスタ値は初期表示に限定し、作業者が確定した値を登録条件とするため |
| `$Ny`はQR値を変更しない | Web版と同じく、返却QRに記録された数量・洗浄方法を表示用・登録分岐用として扱うため |
| 通常QRの入力値0は`SmxNyuSheet` | 洗浄方法0でも`T_シートリーダ`へ実績を残すため |
| `$Ny`の洗浄方法0は同一トランザクション | 終了更新またはシート登録の一方だけが残る状態を防ぐため |
| `$Ny`の洗浄方法0以外は終了更新のみ | Web版の返却登録仕様と処理結果を揃えるため |
| 従来`SmxNyu`も残す | 通常QRの入力値が0以外の場合の既存運用を維持するため |

## 不具合対応

| 項目 | 内容 |
|---|---|
| 現象 | `Soc.YmdSend`で例外が発生すると、別関数の終了ラベルへ復帰する可能性があった |
| 原因 | エラーハンドラが`resume socSend.End`を指定していた |
| 影響範囲 | 山寺入庫、従来SMX入庫・出荷、新しいSMX入庫3通信 |
| 対応 | `resume socYmdSend.End`へ修正 |
| 回避方法 | 修正版を配布するまで、通信が安定した環境で使用し、異常時はDB登録結果を確認する |

### 洗浄方法0でシートリーダ実績が作成されない不具合

| 項目 | 内容 |
|---|---|
| 現象 | 通常QRで洗浄方法に0を入力しても`T_シートリーダ`にデータが作成されない |
| 原因 | 通常QRが新しい`SmxNyuSheet`を通らず、処理分岐もBHTの入力値を受け付けていなかった |
| 影響範囲 | SMX入庫で通常QRを読み取り、洗浄方法0として登録する端末・利用者 |
| 対応 | 洗浄方法入力画面を追加し、入力値0のとき`SmxNyuSheet`を呼ぶよう修正 |
| 回避方法 | 修正版配布までは対象QRの登録結果をDBで確認し、必要に応じて管理者が実績を補完する |

### `$Ny`の終了更新とシート登録が片方だけ残る不具合

| 項目 | 内容 |
|---|---|
| 現象 | `$Ny`の洗浄方法0で、通信や後続登録の失敗により終了日時更新だけが残る可能性がある |
| 原因 | BHTから終了更新とシート登録を別々に実行すると、2処理を一括でロールバックできない |
| 影響範囲 | `$Ny`戻り品QRのうち、洗浄方法0を使用するSMX入庫端末・利用者 |
| 対応 | `SmxNyuSheet`内で終了日時更新とシート登録を同一トランザクション実行し、どちらかが失敗した場合は両方をロールバックする |
| 回避方法 | 修正版配布までは`T_SmxTrc.終了日時`と`T_シートリーダ`の両方を確認する |
